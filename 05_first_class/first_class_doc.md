## Decorator
### 데코레이터는 내가 작성하는 코드의 긍정적인 영향을 엄청나게 증폭시킬 수 있다.

- 파이썬 함수를 작성하는 방법만 알면 데코레이터 사용법을 배울 수 있다.

```py
@somedecorator
def some_function():
    print("Check it out, I'm using decorators!")
```
- 하지만 데코레이터를 작성하는 것은 완전히 다른 기술이다. 그리 간단하지도 않다고 한다. 다음과 같은 내용을 이해해야한다. -> 한번 다시 정리도 하면서 넘어가자.

    - 클로저
        - 아직 이 개념의 필요성을 모르겠다.
        - 클래스를 사용하는게 더욱 명확해보인다.
            - 일반 함수(def)와 익명 함수(lambda)처럼 가볍게 쓸 수 있다는 장점 말고는 아직 모르겠다.
    - 일급 객체
        - 변수에 함수를 할당할 수 있다.
            - `var = func` -> `()`를 안써야한다. 쓰면 함수가 실행이 된다.
        - 다른 함수의 매개변수로 할당할 수 있다.
            - `fucn_1(func_2)` -> 
        - 다른 함수의 반환 값으로 사용할 수 있다.
            - 아래 `outer()`의 반환 값이 `inner` 함수다
                ```py
                def outer():
                    def inner():
                        print('hi')
                ```
        - 런타임 초기화
        
    - 가변 인자
    - 인자 풀기? (argument unpacking)
        - `func(*[1, 3])` -> `func(1, 3)`
        - `*`을 컨테이너 앞에 붙여줌으로써 벗겨준다.
        - 단, `dict`를 사용할 때는 `**`를 사용.
    - 파이썬 소스코드를 불러오는 자세한 과정

> 위 개념들은 이전에 배웠다. (이걸 작성하는 시점이 클로저를 공부한 뒤)
